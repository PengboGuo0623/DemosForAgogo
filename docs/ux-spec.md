# Math Rescue UX Spec

## UX Direction

Math Rescue is a child-friendly, mobile portrait first math demo. The interface
should feel bright, simple, safe, and responsive.

The experience should help a child understand what to do quickly without
reading long instructions.

## Layout Priorities

- Design for phone portrait first.
- Keep the main question large and centered.
- Keep answer choices large, separated, and easy to tap.
- Keep progress visible but not distracting.
- Avoid dense UI, small labels, and adult dashboard patterns.
- Ensure all important content fits on narrow mobile screens.

## Touch Targets

Answer options and primary controls should be large enough for young children.

Recommended minimums:

- answer button height: 64px or larger
- primary button height: 56px or larger
- spacing between answer buttons: 12px or larger

Buttons should look clearly tappable through shape, color, scale, and feedback.

## Screen Flow

### Start Screen

Purpose: invite the child into the rescue mission.

Required elements:

- game title: "Math Rescue"
- friendly rescue visual
- one clear start button
- minimal mission prompt

Avoid long explanations.

### Game Screen

Purpose: answer math questions and see rescue progress.

Required elements:

- rescue scene
- current question
- three answer choices
- progress indicator for 8 questions
- immediate feedback area or animation

The game screen should not feel like a worksheet. Math should be connected to
the rescue action.

### Result Screen

Purpose: celebrate completion.

Required elements:

- completion message
- positive visual celebration
- simple summary, if used
- play again button

Avoid ranking the child harshly. Do not show failure language.

## Feedback Rules

Correct answer feedback:

- immediate
- joyful
- visually clear
- short
- tied to rescue progress

Wrong answer feedback:

- gentle
- brief
- non-shaming
- recoverable
- no loud negative visuals

Use supportive copy such as:

- "Almost!"
- "Try again"
- "So close"

Do not use copy such as:

- "Wrong"
- "Fail"
- "Bad"
- "No!"

## Text Density

Children ages 5-8 may not read fluently. Use minimal text:

- one short instruction at a time
- short button labels
- large numerals
- avoid paragraphs inside the game

Visual guidance should carry the experience.

## Motion and Animation

Animation should make the game feel alive, but not chaotic.

Recommended animation moments:

- answer button press
- correct answer pop
- rescue progress movement
- star or sparkle reward
- result celebration

Avoid long unskippable animations that slow the 60-90 second session.

## Accessibility and Comfort

- Maintain strong contrast between text and background.
- Avoid tiny type.
- Avoid rapid flashing.
- Keep touch targets away from screen edges.
- Respect mobile safe areas where practical.
- Keep audio optional if audio is added later.

# Math Rescue Art Direction

## Visual Goal

Math Rescue should look original, bright, rounded, and friendly. The game should
feel like a playful rescue mission for young children, not a formal worksheet.

The MVP should prioritize clear visual communication over asset complexity.

## MVP Asset Strategy

Use code-drawn visuals first:

- Phaser graphics
- simple geometric characters
- rounded UI panels
- stars, badges, ropes, clouds, paths, number friends, and rescue markers
- lightweight particles or simple shape animations

This keeps the first phase fast, avoids licensing concerns, and makes it easy
to iterate on layout and gameplay.

## Style Principles

- bright but not overwhelming
- rounded corners and soft shapes
- clear silhouettes
- large expressive elements
- simple background depth
- playful rescue theme
- friendly original characters

Use color to guide attention:

- question area should be calm and readable
- answer choices should stand out
- correct feedback should feel warm and celebratory
- wrong feedback should be soft and non-threatening

## Suggested Original Theme Elements

The implementation may use original generic elements such as:

- star friends waiting on small islands
- number buddies stuck behind a soft obstacle
- small animals needing bridge pieces
- a rescue balloon that fills with correct answers
- a progress path with 8 steps

These are starting points, not mandatory assets.

## Prohibited References and Assets

Do not copy Lingokids or any other product's specific:

- characters
- mascots
- costumes
- UI screens
- button shapes that are recognizable as a product identity
- illustrations
- icons
- sound effects
- music
- copywriting
- logo treatment
- brand colors used as a recognizable system
- scene composition

The demo may reference only high-level children's education principles such as
bright visuals, rounded UI, short sessions, low frustration, strong feedback,
and minimal text.

## Typography

Use readable, friendly system fonts for the MVP unless a later phase adds a
licensed font.

Priorities:

- large numerals
- high contrast
- short labels
- no condensed or decorative fonts for core math content

## Animation Feel

Animations should feel soft and responsive:

- bounce on correct answers
- glow or pulse on progress
- gentle shake or nudge for wrong answers
- celebratory particles at session completion

Avoid scary effects, aggressive red error states, or dramatic failure animation.

## Future Asset Upgrade Path

After the MVP is playable, the project can add original bitmap assets if needed.
Any generated or commissioned assets should be stored with clear ownership notes
and checked against the originality rules above.
